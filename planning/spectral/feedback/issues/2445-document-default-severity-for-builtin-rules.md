# document default severity for builtin rules

Spectral comes with builtin rule sets for OpenAPI and AsyncAPI. While the rules themselves are documented [here](https://github.com/stoplightio/spectral/blob/develop/docs/reference/openapi-rules.md) and [here](https://github.com/stoplightio/spectral/blob/develop/docs/reference/asyncapi-rules.md), nowhere does it seem to be documented what the default severity (error/warn/info/off) is for each.

Please add this information to the documentation.
