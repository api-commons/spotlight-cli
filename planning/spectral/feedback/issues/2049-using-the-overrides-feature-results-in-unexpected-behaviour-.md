<details>
<summary>.spectral.yaml</summary>

```yaml
extends: spectral:oas

rules:
  global-version:
    message: "{{property}} path should start with a global version prefix."
    severity: error
    given: "$.paths[*]~"
    then:
      function: pattern
      functionOptions:
        match: "^\/v\\d+(\/|$)"

overrides:
  - files:
      - "test.yaml"
    rules:
      global-version: "off"
```
</details>

1. Given the above spec and spectral config
2. Open test.yaml in VS code or load the directory containing these files in Stoplight Studio
3. In VS code the `global-version` rule is not turned off. In Stoplight Studio, no warnings/errors are shown at all!

**Expected behavior**

The behaviour is as expected using docker (warnings, but no error about `global-version`): `docker run --rm -it -v $(pwd):/tmp stoplight/spectral lint --ruleset "/tmp/.spectral.yaml" "/tmp/test.yaml"`

I would expect the same behaviour in VS Code and Stoplight Studio: the disabled rule is not triggered, but others should continue to run.
