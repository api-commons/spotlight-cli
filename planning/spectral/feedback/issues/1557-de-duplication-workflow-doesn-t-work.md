# De-duplication workflow doesn't work

**Describe the bug**
I’m having trouble getting de-duplication working in a Spectral linter I’m implementing. Some of my rules can generate multiple warnings, but Spectral only shows the first one generated for each rule that fires. I’ve followed the instructions for this from the Spectral docs, but it’s not working. I customized the “path” property of the result to be unique for each warning, using this code, modeled on the code in the docs:

    results.push({
      message: param + ' does not follow lower-kebab-case in path: ' + pathToCheck,
      path: [...rootPath, i, 'name'] // "i" is unique for each warning created.

However, I still only see a single warning per rule firing.

I also implemented a custom “computeFigerprint” function, to see what data it is passed:

    const spectral = new Spectral({
        computeFingerprint: (rule, hash) => {
          let id = String(rule.code);
          if (rule.path && rule.path.length) {
            id += JSON.stringify(rule.path);
          } else if (rule.range) {
            id += JSON.stringify(rule.range);
          }
          if (rule.source) id += rule.source;
          return hash(id);
        },
      });

What I’ve seen is the “rule.path” property passed to computeFingerprint is the standard path used by Spectral when the result doesn’t specify a path. My custom path that I added in the message list is not passed in to computeFingerprint, so only a single warning per rule firing is preserved.



**To Reproduce**

I can't offer a repro case without sending you my custom function and ruleset, which are not yet public. 
To reprocuce yourself, add a custom path to a custom function that produces more than one warning.

**Expected behavior**
When I add two or more unique custom paths to a warnings generated by a custom function, I expect to see all of them reported, not just the first one.

**Environment (remove any that are not applicable):**
 - Library version: 5.5.0
 - OS: Big Sur
