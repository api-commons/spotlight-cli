# Missing  Rule or Code Name in HTML Output Format

Currently, the HTML output format  generated by Spectral does not include the rule names associated with each validation error. While the default console output and JSON reports provide this information, the HTML report only displays the error message, path, and line number.

I'm wondering if there's a specific reason why rule names are excluded from the output HTML format? It would significantly enhance the usability of the HTML report if rule names were included, allowing users to quickly identify which rules are being violated and more easily understand the context of each error.

Why is the rule name already available in the default console output and JSON reports, but not the HTML? Is it a design choice, or an oversight?
